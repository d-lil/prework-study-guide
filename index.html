HTML

  • HTML stands for Hypertext Markup Language
  • The head element contains information about the webpage - metadata
  • The body element represents the visible content shown to the user
  • Title element defines title and is crucial for Search Engine Optimization SEO
  • Style defines CSS styles
  • Base defines base URL
  • Link connects external resource to HTML document
  • Meta defines metadata such as character set, description, keywords, author, and viewport
  • Script embeds client-side scripts in the HTML document
  • Elements listed in other elements are called child elements
  • Body element is the parent element
  • Div creates a division on page
  • Span creates inline division
  • Ul - unordered list, ol - ordered list, li - list
  • Img element contains info about images displayed on webpage
  • Src attribute defines location of image
  • The 'a' element stands for anchor and creates links to the same webpage or other webpages
  • The href attribute points to the URL for the link

CSS

  • Internal style sheet contains CSS rulees for webpage in the head section of HTML file. Rules only apply to that page
  • You can configure CSS classes and IDs to style multiple elements in page code
  • A single change to the CSS rule will apply to all tagged elements
  • External style sheet is most common way to style a webpage
  • Preferred because it allows developers to keep all CSS rules into a separate file
  • When using an external style sheet, link the file using link element placed in HTML's head
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • A period preceding card designates a class sector

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch and staging area, while -A indicates add all changes
  • git commit -m "Description": is a command with a commit message that describes what the commit contains
  • git pull origin main: receives a branch's modifications into local environment, the source of the pull will be in GitHub rep, and which branch in repo
  • git push origin (enter_branch_name_without_parenthesis): pushes the changes made loacally to remote GitHub branch
  • In GitHub repo you must then create a pull request to merge current branch to main branch

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • An 'if' statement is a way to introduce decision-making into the code
  • An 'if' conditional decides if the information is truthy or falsy
  • A Boolean value = true / false
  • Adding 'else if' or 'else' after an 'if' statement will provide the code to perform a different task if conditions to prior statement are not meant or are not truthy
  • === is a strict equality operator that tests if two values are equal
  • An array is a way JS stores a group of related data in a variable
  • A 'for' loop is a way to log each topic from array into console
  • An array can hold strings, numbers, Boolean values, etc which are listed in brackets and divided by commas ["EX", 4, MPLE]
  • Index numbers of items in arrays start at '0' ex- [0, 1, 2, 3] or [0]
  • console.log: shows code working in Chrome DevTools Console tab
  • Adding ++ to a variable (ex. x++) increases it by one each time the code loops
  • To declare a funtion use the function keyword with a unique funtion keyword followed by parenthesis ex- function functionWhatever()
  • After a funtion is declared follow it with { and enter code after that before closing it }
  • If there are a list of conditionals inside curly brackets, finish each conditional with a ;
